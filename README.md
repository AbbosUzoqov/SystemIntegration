Данный проект реализует интеграционное приложение для курса **"Интеграция систем"**.  
Цель работы — синхронизация данных между CMS, базой исторических данных и системой отчётности (Report API).

---

## Функциональность

### Поток данных №1 — синхронизация с CMS
- получает данные из CMS API (с учётом пагинации)
- обновляет и добавляет записи в базу исторических данных
- сохраняет *удалённые* данные (которые CMS больше не отдаёт)

### Поток данных №2 — генерация и отправка CSV отчёта
- формирует CSV на основе базы исторических данных  
- формат CSV:  
  - кодировка: **UTF-8**  
  - разделитель: **";"**
- отправляет сформированный файл в **Report API**

---

## Использованные технологии
- **n8n** — основная интеграционная платформа  
- **DataTable (n8n)** — используется как база исторических данных  
- **CSV** — формат отчётных данных  

---

## Импорт и запуск

### 1. Импорт Workflow в n8n
1. Открыть n8n  
2. Workflow → Import from File  
3. Выбрать `Integration App.json`  

### 2. Запуск
Нажать **Execute Workflow** — n8n автоматически:
- загрузит данные из CMS  
- актуализирует базу исторических данных  
- сформирует CSV  
- отправит CSV в Report API  

---

Система показывает:
- количество деталей  
- количество найденных  
- количество обновлённых  
- итоговую автоматическую оценку
